🚀 Discord Clone API Documentation

Interactive API explorer - test endpoints directly in your browser!

🔐 Authentication Endpoints

POST /api/auth/register - Register new user
POST /api/auth/login - Login user
GET /api/auth/me - Get current user (protected)
GET /api/auth/test-protected - Test protected route

🏢 Server Management Endpoints

POST /api/servers - Create new server
GET /api/servers - Get servers for current user
GET /api/servers/:serverId - Get server by ID
PUT /api/servers/:serverId - Update server
DELETE /api/servers/:serverId - Delete server

📝 Channel Management Endpoints

POST /api/channels - Create new channel
GET /api/channels/server/:serverId - Get channels for a server
GET /api/channels/:channelId - Get channel with messages
DELETE /api/channels/:channelId - Delete channel

💬 Message Endpoints

POST /api/messages/dms - Create a DM
POST /api/messages - Send a message
GET /api/messages/channel/:channelId - Get channel messages
GET /api/messages/dms - Get all DMs for current user
GET /api/messages/dm/:dmId - Get DM messages
GET /api/messages/:messageId - Get single message
PUT /api/messages/:messageId - Update message
GET /api/messages/search/channel/:channelId - Search messages
GET /api/messages/search/dm/:dmId - Search DM messages
DELETE /api/messages/:messageId - Delete message
POST /api/messages/:messageId/reactions - Add reaction
GET /api/messages/:messageId/reactions - Get reactions
DELETE /api/messages/:messageId/reactions - Remove reaction

🔧 Utility Endpoints

GET /health - Health check
GET /api - API information

📝 Instructions

  1. Register a new user or login to get a JWT token
  2. The token will be automatically stored and used for protected routes
  3. Test protected endpoints to verify authentication works
  4. Create servers and channels to test the full functionality
  5. Create and send messages to test message retrieval
  6. Add reactions to test additional features
  7. Check responses for detailed API information

🧪 Testing Workflow

  1. Authentication: Register/Login to get your token
  2. Create Server: Use the server management section to create a new server
  3. Create Channel: Use the server ID from step 2 to create channels
  4. Create Message: Use the channel ID from step 3 to send messages
  5. Fetch Data: Test GET endpoints to retrieve your servers, channels, and messages
  6. Update/Delete:Test PUT and DELETE operations on your resources
